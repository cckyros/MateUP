## 四、Phase 8 详细任务分解

### 任务 8.1：评价列表

**后端需求：**
```
GET /api/players/:id/reviews
Response: {
  data: [
    {
      id, orderId, userId, userName, userAvatar,
      rating, comment, tags[], images[],
      createdAt, playerReply?
    }
  ],
  total, page, pageSize
}
```

**前端改动：**
- `PlayerDetailPage.tsx` — 详情 Tab 切换（简介 / 评价）
- 评价卡片组件复用 `OrderRating.tsx` 的星星逻辑
- 分页加载（滚动加载更多）
- 空状态："暂无评价，成为第一个预约的用户吧"

**预计工作量：** 后端 2h + 前端 3h

---

### 任务 8.2：下单须知

**内容拟定：**
```
预约规则：
1. 预约成功后陪玩师将在 15 分钟内接单
2. 陪玩师超时未接单自动退款
3. 服务开始前可免费取消，开始后按剩余时长退款
4. 如遇陪玩师迟到或中途下线，可申请部分退款
5. 请文明陪玩，尊重陪玩师
```

**前端改动：**
- PlayerDetailPage 底部操作栏加"预约规则"文字按钮
- 点击弹出 Modal 或 BottomSheet
- PaymentPage 确认支付前增加勾选"我已阅读并同意《预约规则》"

**预计工作量：** 前端 1.5h

---

### 任务 8.3：在线时间偏好

**后端需求：**
```
players 表新增字段：
online_hours: JSON, 格式: { "weekdays": "20:00-24:00", "weekends": "全天" }
```

**前端改动：**
- PlayerDetailPage 详情区底部加一行："常在线：周末全天 / 工作日 20:00-24:00"
- 无数据时显示"随时在线"

**预计工作量：** 后端 1h + 前端 1h

---

### 任务 8.4：收藏/关注陪玩师

**后端需求：**
```
POST /api/favorites/:playerId    # 收藏/取消收藏
GET    /api/favorites           # 我的收藏列表
```

**前端改动：**
- PlayerDetailPage 顶部头像区加 ❤️ 按钮（点击切换收藏状态）

- 收藏列表页（可复用 PlayerListPage 布局，过滤条件为收藏列表 API）

**预计工作量：** 后端 3h + 前端 3h
