## 二、Phase 8 — 详情页增强（商品化改造）

> 目标：让陪玩师从"有个详情页"变成"敢下单的商品详情"

### P0 — 影响转化（必须做）

| # | 任务 | 描述 | 涉及页面 | 前后端 | 工作量 |
|---|------|------|---------|--------|--------|
| 8.1 | **评价列表** | 详情页底部 Tab：评价列表（文字+评分+晒图+时间），支持分页加载 | PlayerDetailPage | 需后端 `GET /api/players/:id/reviews` | 后端 2h + 前端 3h |
| 8.2 | **下单须知** | 底部固定"预约规则"按钮，弹窗展示退款/迟到/取消政策；PaymentPage 支付前勾选服务协议 | PlayerDetailPage, PaymentPage | 纯前端 | 前端 1.5h |
| 8.3 | **在线时间偏好** | 详情页显示"常在线：周末全天 / 工作日 20:00-24:00" | PlayerDetailPage | 需后端 `player.online_hours` 字段 | 后端 1h + 前端 1h |

#### 8.1 评价列表 — 详细设计

**后端 API：**
```
GET /api/players/:id/reviews
Response: {
  data: [
    {
      id, orderId, userId, userName, userAvatar,
      rating, comment, tags[], images[],
      createdAt, playerReply?
    }
  ],
  total, page, pageSize
}
```

**前端改动：**
- `PlayerDetailPage.tsx` — 详情 Tab 切换（简介 / 评价）
- 评价卡片组件复用 `OrderRating.tsx` 的星星逻辑
- 分页加载（滚动加载更多）
- 空状态："暂无评价，成为第一个预约的用户吧"

#### 8.2 下单须知 — 内容

```
预约规则：
1. 预约成功后陪玩师将在 15 分钟内接单
2. 陪玩师超时未接单自动退款
3. 服务开始前可免费取消，开始后按剩余时长退款
4. 如遇陪玩师迟到或中途下线，可申请部分退款
5. 请文明陪玩，尊重陪玩师
```

**前端改动：**
- PlayerDetailPage 底部加"预约规则"文字按钮
- 点击弹出 BottomSheet
- PaymentPage 确认支付前勾选"我已阅读并同意《预约规则》"

#### 8.3 在线时间偏好

**后端字段：** `players.online_hours: JSON` → `{ "weekdays": "20:00-24:00", "weekends": "全天" }`

**前端：** 详情区底部显示在线时段，无数据时显示"随时在线"

## 六、建议执行顺序

```
第一批（核心体验，~7h，纯前端可立即做）:
  ① 下单须知 & 服务协议     1.5h    [8.2]
  ② 订单进度时间轴           2h     [9.2]
  ③ 骨架屏加载态             2h     [9.3]
  ④ 首页 Banner 轮播         2h     [9.1]

第二批（粘性提升，~7h）:
  ⑤ 标签体系增强             1.5h   [9.5]
  ⑥ 未读消息红点             2h     [9.6]
  ⑦ 下拉刷新 & 加载更多      3h     [9.4]

第三批（转化提升，~8h，部分需后端）:
  ⑧ 评价列表                 5h     [8.1]
  ⑨ 评价晒图                 3h     [8.6]

第四批（差异化，按需选做）:
  ⑩ 优惠券系统               4h     [10.2]
  ⑪ 陪玩师等级徽章           2h     [10.3]
  ⑫ 路由懒加载               2h     [技术优化]
```

**预计总工期（不阻塞情况下）：**
- 第一批 + 第二批 ≈ 3 天
- 第三批 ≈ 2 天
- 第四批按需
